Composition Rules

  • Use a desktop 12-column grid: left/center identity spans the larger area, while the right side holds layered product previews.
  • Place a curved learning path on the far left with small milestone nodes to echo the first sketch without making the page look hand-drawn.
  • Add connected circular route nodes across the upper-right area to reference the second sketch's learning progression idea.
  • Keep generous whitespace around the wordmark so the hero feels intentional and premium.
  • On mobile, collapse into this order: feature rail, headline, supporting line, CTAs, product preview, feature keyword band.

Course Explanation Flow

The last course preview should communicate course viability through a tiny adaptive-learning interaction. The user can flag confusion from inside the course UI, MentorPet responds with a focused explanation, and the course records the outcome.

UI Moment Required Detail
Student message Show a compact message such as “I didn't understand a basic concept about Newton's 2nd Law.”
MentorPet response Show a short explanation card connected to the lesson topic, written as study support rather than open-ended chat.
Side status Show a small status chip or report message reading “Done” after the explanation is generated or saved.
Course note Add a visible personal note to the course, tied to the lesson and topic, so the student can revisit the explanation later.

In the hero, this should be visualized as a product preview card, not a full chat window. The point is to show that Mentoria courses can become more personal when a student marks a concept as unclear.

Feature Keyword Band

The lower hero band should use large, scannable keywords rather than full sentences. Include these terms: SAT, IELTS, Profile, Course Roadmap, Books, and Courses.

The band should sit low enough to feel like the sketch's bold bottom layer, while still leaving a visible hint of the next section below the hero.

Visual Direction

  • Use an off-white background, charcoal text, muted green accent, and soft neutral borders.
  • Ignore the rough sketch colors entirely; preserve layout intent, not marker color.
  • Avoid a centered generic hero. The layout should feel asymmetric, product-led, and spacious.
  • Use a premium sans-serif such as Geist, Satoshi, Outfit, or a similar clean UI font if available in the app.
  • Avoid making MentorPet babyish. The widget should feel friendly and credible for students in grades 8-11.

Interaction And Motion

  1. Feature rail fades in first.
  2. Mentoria Hub wordmark slides/fades in next.
  3. Path nodes and orbit nodes draw in with a subtle stagger.
  4. Product preview cards float in with a slight stagger.
  5. MentorPet can blink or wave subtly inside the widget.

Keep all animation transform/opacity based. Product cards may lift gently on hover. Avoid needy companion copy such as “I missed you” or guilt-based absence messaging.
