Context Guidelines

Repository note for context-related artifacts.

Rule: everything context-related in this repository should be saved as an HTML file with a .html extension, not as Markdown with a .md extension.

Recommended Practice

  • Place context files in the context/ directory.
  • Use semantic HTML sections, headings, lists, and tables so the file is easy to read in a browser and easy for tools to parse.
  • Include source information and generation date when the context is derived from another file.
  • Link related context files together when useful.
  • Avoid creating Markdown context files unless a future user explicitly overrides this convention.
